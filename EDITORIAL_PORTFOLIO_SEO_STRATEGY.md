# Editorial Portfolio SEO Strategy

## Purpose

This file defines the content, SEO, and layout strategy for the portfolio gallery/project pages on kchernek.com.

The goal is to make each portfolio gallery feel like a polished editorial project page instead of a basic image gallery. Each page should remain image-first, but include enough written context to improve visual pacing, user experience, and SEO.

This strategy should guide future Claude Code, Codex, or manual website updates.

---

# Core Direction

Do not treat the portfolio as a generic blog.

Do not mass-produce SEO articles just to add keywords.

Instead, treat each gallery as a real project or editorial feature.

Each gallery page should function as:

- a visual portfolio gallery
- a project case study
- an editorial photo story
- a search-friendly landing page
- a credibility signal for future clients, models, agencies, or brands

The written copy should support the images, not overpower them.

---

# Strategic Rationale

Most photography portfolio pages are image-heavy but text-light.

That creates several issues:

- Google has limited context for what the page is about
- images are often not supported by useful written information
- galleries can feel like image dumps
- visitors may scroll quickly without understanding the project
- individual shoots do not build much search relevance
- there are fewer opportunities for internal links and service-specific CTAs

Adding concise project-based text solves this without forcing a traditional blog format.

This approach allows each gallery to naturally include:

- shoot type
- location
- styling direction
- creative intent
- lighting approach
- production notes
- service relevance
- relevant keywords
- internal links
- soft calls to action

---

# What The Pages Should Feel Like

Each gallery page should feel like:

- a premium photography portfolio feature
- a fashion/editorial project page
- a campaign presentation
- a concise creative case study
- a visually paced magazine-style layout

The pages should not feel like:

- a generic blog post
- an SEO article
- a keyword-stuffed landing page
- a stock template
- a basic masonry image dump
- a long personal journal entry
- an over-explained tutorial

---

# Writing Style

The written copy should feel:

- polished
- editorial
- confident
- specific
- understated
- visual
- professional
- concise

The tone should support a photography portfolio and booking website.

Avoid generic photography cliches such as:

- "capturing moments"
- "telling stories through my lens"
- "bringing visions to life"
- "stunning images"
- "unforgettable memories"
- "timeless memories"
- "elevated visuals"
- "visual storytelling" unless specifically justified
- "creating magic"
- "picture-perfect"

Prefer concrete language about:

- light
- location
- wardrobe
- styling
- composition
- pacing
- production choices
- direction
- texture
- tone
- mood
- use case
- visual intent

The copy should sound like a thoughtful photographer describing the work, not like an SEO agency writing filler text.

---

# SEO Philosophy

SEO should be natural, restrained, and grounded in the actual project.

Do not force keywords.

Do not repeat the same search phrase excessively.

Do not invent shoot details to satisfy SEO.

Do not turn gallery pages into long-form blog posts unless specifically requested.

The best SEO usage should appear naturally in:

- page title
- H1
- intro paragraph
- image alt text
- filename patterns
- section headings
- meta description
- internal links
- closing CTA

The page should help search engines understand:

- what type of photography this is
- where the shoot took place
- what service category it supports
- what kind of client or model it is relevant to
- how the work fits into the broader portfolio

# Ideal Gallery Page Structure

Each gallery page should use an editorial rhythm rather than one uninterrupted image grid.

A typical page may include:

1. Project hero section
2. Opening image group
3. Introductory project text
4. Second image group
5. Mid-page creative or production context
6. Third image group
7. Optional production note
8. Final image group or closing hero image
9. Soft call-to-action
10. Related portfolio or service links

The exact number of text blocks should depend on the number of images and the amount of useful shoot information available.

Default guidance:

- Small gallery, 6-10 images: 1-2 text blocks
- Standard gallery, 11-20 images: 2-3 text blocks
- Large gallery, 21+ images: 3-4 text blocks maximum

Never add text just to fill space.

---

# Text Block Types

## Project Intro

Length: approximately 75-150 words.

Purpose:

- introduce the shoot
- establish genre, location, and visual direction
- naturally include relevant SEO terms
- set the tone before the viewer sees the full gallery

The intro may reference:

- shoot type
- location
- concept
- mood
- styling
- model/client goal
- service category

Example purpose:

"This section explains what the project is and why it belongs in the portfolio."

---

## Mid-Page Context Block

Length: approximately 100-250 words.

Purpose:

- add context once the viewer has seen some images
- describe creative direction or production choices
- break up the visual flow
- help the gallery feel intentional

This section may reference:

- lighting approach
- styling choices
- wardrobe
- location use
- model direction
- creative constraints
- visual pacing
- composition choices
- behind-the-scenes context

This should not become a tutorial unless the page specifically calls for it.

---

## Optional Production Note

Use only when the shoot has useful details worth mentioning.

Possible topics:

- lighting
- location
- styling
- model direction
- gear/lens approach
- post-production
- client use case
- lessons from the shoot

Keep this section concise.

---

## Closing Note / CTA

Length: approximately 50-125 words.

Purpose:

- wrap the project cleanly
- connect the gallery to a relevant service
- guide visitors toward inquiry or related work

The CTA should be soft and specific.

Examples:

- "For model digitals and portfolio-building sessions in Dallas, visit the model portfolio page."
- "For restaurant photography inquiries, use the contact page to start a project."
- "For swimwear, editorial, or commercial lifestyle shoots, view related work or reach out."

Avoid aggressive sales language.

---

# Image Layout Strategy

Do not show all images in one uninterrupted grid by default.

Group images into sections.

Determine the layout based on the number of images for the specific page.

Possible layout modules:

- full-width hero image
- 2-image editorial row
- 3-image grid
- asymmetric image grid
- mixed portrait/landscape layout
- horizontal strip
- carousels
- final full-width closing image
- related project cards

The layout should create visual rhythm.

Avoid repetitive patterns such as:

- 3 images, 3 images, 3 images, 3 images forever
- one image per row for the entire page
- a dense masonry dump with no breathing room
- tiny thumbnails that reduce image impact

The page should feel more like an editorial feature than a file browser.

---

# Desktop Layout Guidance

On desktop:

- use generous whitespace
- allow strong hero images
- use multi-image rows where appropriate
- vary the rhythm between full-width images and grids
- avoid making every image the same size
- keep text blocks clean and narrow enough to read comfortably
- let strong images breathe
- avoid visual clutter

---

# Mobile Layout Guidance

On mobile:

- images should remain large enough to appreciate
- complex desktop grids should stack cleanly
- text should not interrupt too frequently
- carousels may be used for grouped supporting images
- avoid awkward crops
- avoid tiny multi-column image grids
- maintain a smooth scroll experience

If carousel functionality is used, it should be feel seamless, accessible, and not required for the entire gallery.

---

# Image SEO Rules

Every important image should have useful metadata.

Use:

- descriptive filenames
- accurate alt text
- relevant captions only when useful
- proper image dimensions
- lazy loading for non-critical images
- priority loading for hero images

Alt text should describe the actual image.

Do not keyword-stuff alt text.

Good alt text:

- "Studio portrait of a model in black-and-white editorial lighting"
- "Dallas swimwear editorial portrait with western-inspired styling"
- "Restaurant cocktail photographed with warm backlight and marble bar texture"

Bad alt text:

- "Dallas photographer best Dallas photographer Dallas model photographer"
- "Amazing stunning beautiful portrait photo"
- "Image 001"

---

# Filename Guidance

Image filenames should be clean, descriptive, and consistent.

Preferred pattern:

```txt
project-name_shoot-type_location_sequence.jpg
```

Examples:

```txt
cowgirl-swimwear_dallas-editorial_001.jpg
studio-black-white_model-portrait_dallas_001.jpg
restaurant-cocktail_photography_dallas_001.jpg
```

Do not use camera export filenames such as:

```txt
DSC08421.jpg
IMG_3827.jpg
untitled-edit-final-final.jpg
```

# Project Page SEO Elements

Each gallery page should include:

## Page Title

A concise, search-aware title that still feels premium.

Examples:

```txt
Dallas Swimwear Editorial - Cowgirl Swimwear
Studio Model Test Shoot in Dallas
Black and White Editorial Portraits
Restaurant Photography in Dallas
```

## URL Slug

Use clean, readable slugs.

Examples:

```txt
/portfolio/cowgirl-swimwear.html
/portfolio/studio-black-white.html
/portfolio/editorial.html
/portfolio/restaurant-photography.html
```

## H1

Usually close to the project title.

## Meta Description

Approximately 140-160 characters.

Should describe the page clearly and naturally.

Example:

```txt
A Dallas swimwear editorial project featuring western-inspired styling, natural light, and portfolio-focused imagery by Keith Chernek.
```

## Internal Links

Each project should link naturally to relevant pages, such as:

- contact page
- main portfolio page
- related portfolio galleries
- model digitals page
- restaurant photography page
- swimwear/editorial service page
- commercial photography page

Only add links where they make sense.

# Use of shoot-info-template.txt

Each gallery folder may contain a `shoot-info-template.txt` file.

Use this file as the source of truth for project-specific placeholder copy.

The file may contain details such as:

- model
- location
- shoot type
- date
- styling
- lighting
- gear
- concept
- notes
- intended use
- keywords

If details exist, use them naturally.

If details are missing, do not invent them.

Use clearly marked placeholders where needed.

Example placeholder:

```txt
[Placeholder intro: add a short description of the shoot concept, location, styling direction, and intended portfolio use.]
```

---

# Placeholder Copy Rules

Placeholder copy should be useful but clearly replaceable.

It should not pretend to know facts that have not been provided.

Good placeholder:

```txt
[Placeholder: This section will describe the shoot concept, location, and creative direction once final notes are added.]
```

Bad placeholder:

```txt
Shot during golden hour in downtown Dallas, this campaign was created for a luxury swimwear brand.
```

Only write specifics like that if they are actually provided.

---

# Automation Rules

Automation may be used for:

- draft copy
- image grouping suggestions
- metadata suggestions
- alt text drafts
- filename suggestions
- internal link recommendations
- layout scaffolding
- SEO title and meta description drafts

Automation should not be used for:

- unsupervised publishing
- inventing project facts
- fake case studies
- fake client outcomes
- keyword stuffing
- mass blog generation
- replacing final human review

All final page copy should be reviewed before publishing.
